## 2. 色彩系统

**主色调**：暖橙 `#F97316` — 传达温暖、活力和宠物亲和

| 角色 | 颜色 | 用途 |
|------|------|------|
| primary | `#F97316` | 按钮、链接、选中态、标签 |
| primary-hover | `#EA580C` | hover/active 状态 |
| primary-light | `#FFF7ED` | 浅色背景、卡片左边框 |
| sidebar | `#1E293B`（深灰） | 侧边栏背景 |
| success | `#16A34A`（绿色） | 成功状态、药品/库存模块 |
| danger | `#EF4444`（红色） | 删除、库存预警 |
| info | `#2563EB`（蓝色） | 信息卡片、营收数据 |

## 4. 路由设计

```
/login                          → 登录页
/                               → 重定向到 /dashboard
/dashboard                      → 根据角色渲染不同仪表盘
/customers                      → 客户列表
/customers/:id                  → 客户详情（含宠物列表 + 就诊历史）
/consultation                   → 挂号列表（按 status 分 Tab）
/consultation/:id               → 就诊详情（接诊 → 诊断 → 处方流程）
/pharmacy                       → 药品列表 + 批次入库
/billing                        → 账单列表（状态筛选）
/billing/:id                    → 账单详情 + 收费项明细 + 结账
/hospitalization                → 住院列表（状态筛选）
/hospitalization/:id            → 住院详情 + 护理记录录入/历史
/boarding                       → 寄养列表
/boarding/:id                   → 寄养详情（含动态费用）
/accounts                       → 账号管理（管理员专属）
/403                            → 权限不足
```

**路由守卫** `ProtectedRoute`：检查 `isAuthenticated()` + `user.role ∈ allowedRoles`，不满足跳 `/login` 或 `/403`。

## 6. 状态管理方案

### 6.1 Zustand Store（客户端状态）

```typescript
// authStore — 登录状态
interface AuthState {
  user: User | null;
  token: string | null;
  login(username, password): Promise<void>;
  logout(): void;
  isAuthenticated(): boolean;
}

// workflowStore — 就诊流程跨页面状态
interface WorkflowState {
  currentVisitId: number | null;
  currentStep: 'register' | 'accept' | 'diagnosis' | 'prescription' | 'billing';
  setStep(step): void;
  reset(): void;
}
```

### 6.2 React Query Hooks（服务端状态）

| Hook | 后端接口 | 缓存策略 |
|------|----------|----------|
| `useCustomers(id?)` | `GET /api/customers[/:id]` | staleTime 5min |
| `useVisits(status?)` | `GET /api/consultation/visits?status=` | staleTime 1min |
| `useVisitDetail(id)` | `GET /api/consultation/visits/:id` | refetch on mount |
| `useMedicines()` | `GET /api/pharmacy/medicines` | staleTime 10min |
| `useBatches(stockLow?)` | `GET /api/pharmacy/batches` | staleTime 5min |
| `useBills()` | `GET /api/billing/bills` | staleTime 2min |
| `useBillDetail(id)` | `GET /api/billing/bills/:id` | refetch on mount |
| `useHospitalizations(status?)` | `GET /api/hospitalization?status=` | staleTime 2min |
| `useHospDetail(id)` | `GET /api/hospitalization/:id` | refetch on mount |
| `useBoardings()` | `GET /api/boarding` | staleTime 5min |
| `useBoardingDetail(id)` | `GET /api/boarding/:id` | refetch on mount |
| `useWards()` | `GET /api/wards` | staleTime 5min |
| `useCustomerHistory(id)` | `GET /api/customers/:id/history` | refetch on mount |

**Mutation 后自动 invalidate 相关 query**，保证数据一致性。

## 7. 关键业务流程

### 7.1 就诊主线（核心）

```
挂号 → 接诊 → 诊断 → 开处方 → 生成收费项 → 结账
```

- 就诊列表页按状态分 Tab
- 详情页顶部展示当前步骤进度（Stepper/Badge）
- 每个步骤对应一个操作区块，完成后进入下一步
- `workflowStore` 跟踪当前 visit_id 和步骤

### 7.2 住院流程

```
已诊断就诊 → 分配空闲笼位 → 转入住院 → 每日护理记录 → 出院（自动计费）
```

### 7.3 寄养流程

```
选择空闲寄养笼 → 登记宠物 → 结束时释放笼位 + 计算费用
```

## 8. Axios 拦截器

```typescript
// request 拦截器
axios.interceptors.request.use(config => {
  const token = useAuthStore.getState().token;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// response 拦截器
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      useAuthStore.getState().logout();
      window.location.href = '/login';
    }
    if (error.response?.status === 403) {
      window.location.href = '/403';
    }
    toast.error(error.response?.data?.detail || '请求失败');
    return Promise.reject(error);
  }
);
```

## 10. 实施顺序
